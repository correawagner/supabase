The `dblink` extension allows you to connect to other PostgreSQL databases from within a database.

## Overview

The `dblink` is an extension which allows to connect to other PostgreSQL database within a database. With `Foreign Data Wrappers (FDW)` you can uniquely define a remote foreign server in order to access its data.

## Usage

`dblink` is used To connect to other PostgreSQL databases from within a database.

### Test connection of database

```
SELECT dblink_connect('host=localhost user=postgres password=enjoy dbname=postgres');
```

### Returned values

The function returns the row(s) produced by the query. Since dblink can be used with any query, it is declared to return record, rather than specifying any particular set of columns.

```
SELECT *
    FROM dblink('dbname=mydb options=-csearch_path=',
                'select proname, prosrc from pg_proc')
      AS t1(proname name, prosrc text)
    WHERE proname LIKE 'bytea%';

```

## Some instance of `dblink`

`dblink` executes a query in a remote database.<br/>
`dblink_connect` opens a persistent connection to a remote database. <br/>
`dblink_disconnect` closes a persistent connection to a remote database.<br/>
