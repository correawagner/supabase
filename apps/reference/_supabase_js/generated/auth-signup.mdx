## Notes

- By default, the user will need to verify their email address before logging in. To disable this, you can toggle off the "Confirm email" setting in [your project](https://app.supabase.com/project/_/auth/settings).
- The "Confirm email" setting determines if users need to confirm their email address after signing up.
  - If "Confirm email" is turned on, a `user` is returned but `session` will be null.
  - If "Confirm email" is turned off, both a `user` and a `session` will be returned
- When the user confirms their email address, they will be redirected to the [`SITE_URL`](https://supabase.com/docs/reference/auth/config#site_url) by default. You can modify your `SITE_URL` or add additional redirect urls in [your project](https://app.supabase.com/project/_/auth/settings).
- If signUp() is called for an existing confirmed user:
  - If "Confirm email" is enabled in [your project](https://app.supabase.com/project/_/auth/settings), an obfuscated / fake user object will be returned.
  - If "Confirm email" is disabled, an error with a message "User already registered" will be returned.
- To fetch the currently logged-in user, refer to [`getUser()`](/docs/reference/javascript/next/auth-getuser).

## Examples

### Sign up.

```js
const { data, error } = await supabase.auth.signUp({
  email: 'example@email.com',
  password: 'example-password',
})
```

### Sign up with additional user meta data.

```js
const { data, error } = await supabase.auth.signUp({
  email: 'example@email.com',
  password: 'example-password',
  options: {
    data: {
      first_name: 'John',
      age: 27,
    },
  },
})
```
